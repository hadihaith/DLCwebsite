How This Works:

  1. Each event has 6 secret numbers (3 for start, 3 for end attendance)
  2. These numbers are randomly generated when the event is created
  3. The QR codes encode URLs containing these secret numbers
  4. Members can display these QR codes for attendees to scan
  5. After scanning, attendees enter their 6-digit attendance code
  6. The system verifies the code and marks their attendance

Testing Steps:

  1. Create a new event in the Django admin or through the portal
  2. The event will automatically get random secret numbers
  3. Go to the event detail page as a member
  4. Click "Get attendance QR code"
  5. Choose whether to show the start or end QR
  6. Scan with your phone or click the URL directly
  7. Enter an attendance code from the event
  8. Verify that the correct boolean (present_start or present_end) is set

  • QR API: Google Charts API (no libraries needed)
  • URL Pattern: /events/verify/{start|end}/{num1}/{num2}/{num3}/
  • Secret Generation: 6-digit random numbers (100000-999999)
  • Storage: secret_start_a/b/c and secret_end_a/b/c fields on Event model
  • Security: Member-only access to QR generation page
